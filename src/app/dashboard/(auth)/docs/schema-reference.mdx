## `gibson.mission.v1.JoinNodeConfig`

JoinNodeConfig blocks until every node ID in `wait_for` has completed (success or final failure), then merges their results per `strategy`. JOIN is a first-class noun separable from PARALLEL — a JOIN can merge results from non-parallel branches.  Spec: mission-verb-noun-registry Requirement 7.

| Field | Type | Tag | Description |
|---|---|---|---|
| `wait_for` | `repeated string` | 1 | wait_for lists the upstream node IDs whose completion is required before this JOIN runs. Must be non-empty; submit-time validation rejects empty wait_for. |
| `strategy` | `gibson.mission.v1.MergeStrategy` | 2 | strategy selects how upstream results are combined. |
| `aggregator` | `string` | 3 | aggregator carries a CEL expression used when strategy is MERGE_STRATEGY_CUSTOM. The expression sees `sources` (a map from node ID to that node's result) and returns the merged value. Empty when strategy is not CUSTOM. |

## `gibson.mission.v1.ParallelNodeConfig`

ParallelNodeConfig contains configuration for parallel nodes. PARALLEL fans out to its sub-nodes concurrently, capped by max_concurrency. Sibling failures are isolated (one failing sub-node does not cancel its siblings). Spec: mission-verb-noun-registry Requirement 6.

| Field | Type | Tag | Description |
|---|---|---|---|
| `sub_nodes` | `repeated gibson.mission.v1.MissionNode` | 1 | SubNodes contains the nodes to execute in parallel |
| `max_concurrency` | `int32` | 2 | MaxConcurrency limits the number of concurrent executions (0 = unlimited) |

## `gibson.mission.v1.PluginNodeConfig`

PluginNodeConfig contains configuration for plugin nodes. PLUGIN = multi-method provider keyed by `plugin_name + method`. Distinct from TOOL: a plugin advertises several callable methods behind one component identity. The executor selects the named method and dispatches `params` as the call payload.

| Field | Type | Tag | Description |
|---|---|---|---|

| `method` | `string` | 2 | Method is the plugin method to call |
| `params` | `map<string, string>` | 3 | Params contains the method parameters |
| `max_tokens_per_call` | `int32` | 4 | max_tokens_per_call overrides MissionConstraints.max_tokens_per_call for this node only. 0 means unlimited; absence means cascade from the mission level. |
