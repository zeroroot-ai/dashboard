Gibson is **zero-day.ai's** AI agent platform for security work. You
build agents, tools, and plugins; Gibson runs them as missions,
captures everything they discover into a knowledge graph, and gives
your team a dashboard to monitor, replay, and act on findings.

These docs are for builders — engineers and security teams using
Gibson to ship autonomous workflows.

## What you build

Gibson exposes three component kinds, each suited to a different shape
of work:

| Component | Use it when… |
|---|---|
| **Agent** | You want autonomous reasoning. Agents pick what to do next, call tools, query the knowledge graph, and submit findings. They run inside a mission. |
| **Tool** | You have a stateless capability with a fixed input/output contract — a port scan, an HTTP probe, a CVE lookup. Tools are called by agents. |
| **Plugin** | You need a long-running, stateful integration — a Shodan client, a SIEM connector, a custom database. Plugins hold credentials and are called by tools. |

You write components against the **Gibson SDK** (Go), register them
with `gibson component register`, and they connect to your tenant on
`api.zero-day.ai`.

## What Gibson does for you

- **Mission orchestration** — you describe work as a DAG of nodes
  (agents, tools, plugins, conditions, parallel branches). Gibson runs
  it, with pause/resume and checkpoint replay.
- **Knowledge graph** — every host, port, service, finding, and attack
  pattern your agents see is stored in a per-tenant graph. Future
  agents query it before re-doing work.
- **Three-tier memory** — working memory inside one agent invocation;
  mission memory shared by every node in the run; long-term memory that
  persists across missions.
- **LLM slot routing** — agents declare slots (`primary`,
  `summarizer`, …); your tenant's policy maps each slot to a real
  model. Bring your own provider keys.
- **Secrets management** — bind credentials to plugins; rotate without
  redeploying.
- **Findings + audit** — every discovery is structured, severity-tagged,
  and tied back to the mission, agent, and target that produced it.
- **Observability** — Langfuse-backed traces, cost/latency metrics,
  per-mission replay.
